# ContainGo Threat Model

## Assets

- Workload identity and private keys delivered through the SPIFFE Workload API.
- OPA authorization policy and quarantine data.
- Risk scores, security events, incidents, and immutable audit records.
- Protected API data.
- Administrative release and reset operations.

## Trust boundaries

### Workload to API Gateway

The client must present a valid SPIRE-issued X.509-SVID. The Gateway accepts only the exact client identities configured for external and internal routes. A caller cannot select its identity with an HTTP header.

### API Gateway to Protected API

The Protected API accepts only the API Gateway SPIFFE ID. Kubernetes NetworkPolicy also allows inbound port 8080 only from the Gateway pod.

### API Gateway to Control Plane

Only the API Gateway identity can submit security events. Dashboard and democtl identities can read administrative state and perform permitted actions but cannot impersonate the Gateway event source.

### Control Plane to API Gateway

Only the Control Plane identity can update quarantine data through the Gateway's internal endpoint.

### democtl to Report Client

Only democtl can modify Report Client mode. The Report Client's control port is restricted by both SPIFFE authorization and NetworkPolicy.

## Threats and controls

| Threat | Control |
|---|---|
| Forged identity header | Identity comes exclusively from a verified X.509-SVID |
| Stolen long-lived certificate file | Private keys are supplied by SPIRE and rotate automatically |
| Calling a service directly to bypass OPA | Exact peer identity checks and default-deny NetworkPolicies |
| Client-selected risk points | Risk contributions and points are generated by server code |
| Replaying a trusted event | Unique request IDs and duplicate-event rejection |
| Unauthorized release | Control Plane administration requires dashboard or democtl SPIFFE identity |
| Container privilege escalation | Non-root user, no privilege escalation, read-only filesystem, dropped capabilities, RuntimeDefault seccomp |
| Kubernetes API token theft | Application pods set `automountServiceAccountToken: false` |
| Lost quarantine after restart | Startup reconciliation restores risk state and OPA quarantine data from SQLite |
| Incomplete incident explanation | Complete current-cycle contribution reconstruction before incident creation |

## Deliberate local-demo limitations

- The kind cluster has one control-plane node and is not highly available.
- SQLite uses a single local hostPath volume.
- The Dashboard is HTTP on loopback through `kubectl port-forward`; its backend connection is SPIFFE mTLS.
- Images use the local `:dev` tag and are loaded directly into kind.
- Secrets, external backups, multi-cluster federation, and production ingress are outside this reference implementation.

These limitations are appropriate for a free local demonstration but must be redesigned for production deployment.
