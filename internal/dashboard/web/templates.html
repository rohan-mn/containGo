SPIRE trust foundationSeven X.509-SVID workload identities · 30-minute TTL · automatic rotation
01

Credentialless identity

No application private keys or static client certificates are stored in containers.

02

Exact peer authorization

Each internal TLS connection authorizes the expected SPIFFE ID, not merely a trusted CA.

03

Policy as code

OPA determines whether a workload identity may call a specific method and path.

04

Behavioral containment

Denied and anomalous requests contribute risk; the Control Plane automatically quarantines at 70.

05

Network isolation

Default-deny NetworkPolicies permit only required pod-to-pod communication paths.

06

Forensic evidence

SQLite retains event contributions, complete incident reasons and administrator audit records.
